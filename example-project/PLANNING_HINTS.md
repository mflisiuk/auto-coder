# PLANNING_HINTS.md

## Repo Conventions

- prefer existing command names over introducing aliases
- if the repo already uses `run`, do not invent `runs`
- keep pagination style consistent with the repo, e.g. `--limit/--offset` instead of cursor-based pagination if that is the local norm

## Planning Guidance

- prefer backend-first task slices before UI polish
- keep each task small enough for 1-3 work orders
- when in doubt, reuse existing folder and naming patterns instead of creating new structure
