# Example Project Input Pack

This directory shows the level of detail that should pass `auto-coder` input validation.

Use it as the reference for:

- how specific `ROADMAP.md` should be
- how concrete `PROJECT.md` should be
- what belongs in `CONSTRAINTS.md`
- when `ARCHITECTURE_NOTES.md` is useful

This is not a generated backlog. It is the human-prepared input contract for the manager.
